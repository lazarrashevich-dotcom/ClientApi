Текстовое описание веб-сервиса Client Management API
1. Назначение системы
Client Management API — это рекомендательная система управления клиентскими профилями, основанная на бизнес-правилах. Система автоматически анализирует данные клиента и формирует персонализированную рекомендацию по условиям обслуживания.
2. Целевая аудитория
Система предназначена для менеджеров банков и финансовых организаций, которым необходимо быстро оценивать клиентский профиль и принимать решения об условиях обслуживания на основе объективных критериев.
3. Основные функции
API предоставляет следующие функции: получение списка всех клиентов (GET /api/Clients); получение клиента по идентификатору (GET /api/Clients/{id}); добавление нового клиента с автоматическим расчётом категории и риска (POST /api/Clients); анализ клиентского профиля с возвратом рекомендации без сохранения (POST /api/Clients/analyze).
4. Архитектурные особенности
Сервис разработан на платформе ASP.NET Core 8 с использованием паттерна Clean Architecture. Бизнес-логика вынесена в отдельный слой Rules (BalanceRule, ActivityRule, RiskRule, ScoreRule). Документация генерируется автоматически через Swashbuckle (Swagger UI). Хранилище данных реализовано в оперативной памяти (in-memory). Развёртывание выполнено в облачном сервисе Amvera через Docker-контейнер.
 
5. Инструкция по запуску
Для локального запуска необходим .NET 8 SDK. Команда запуска: dotnet run --project ClientApi. После запуска Swagger UI доступен по адресу http://localhost:8080/swagger. Для облачного развёртывания используется Dockerfile в корне проекта. Приложение развёрнуто по адресу: http://amvera-student27042026-run-clientapir.amvera.io
